# Tamagotchi

Данный проект представляет из себя мини-игру «Тамагочи» с GUI. Проект составлен при помощи Java FX и графического редактора Photoshop для создания необходимых текстур объектов. Версия Java 1.8. 
Сборка проекта производилась при помощи системы контроля версий Maven. Инструкция по сборке будет далее в данном документе.

Жизненный цикл приложения: 
1)	«Запуск и создание питомца» - пользователь открывает приложение, если у него еще нет живого питомца, он заводит его, выбирая один из представленных типов питомца.
2)	«Уход за питомцем» - пользователь может управлять питомцем, кормить его и повышать его настроение.
3)	«Смерть питомца» -  последствием неправильного ухода будет смерть питомца, после чего в течение определенного времени нельзя создать нового питомца.

Функционал игры:
1)	Пользователь может выбрать один из предложенных вариантов питомцев.
2)	Пользователь может кормить и играть с питомцем.
3)	При долгом отсутствии еды или игры, соответствующая шкала на GUI снижается.
4)	При закрытом приложении, учет пройденного без ухода времени производится при следующем запуске приложения.
5)	Питомец способен расти. Через определенный промежуток времени питомец вырастает от состояния новорожденного (при создании), до взрослого питомца.
6)	При выходе за границы диапазона допустимых значений «сытости» и «веселья» программа сообщает пользователю об этом («Питомец не хочет есть», «Питомец скучает по вам» и т.д.).
7)	Питомец может передвигаться по игровому полю (комнате) при помощи клавиш UP, DOWN, RIGHT, LEFT.
8)	Для каждого типа питомца существует свой тип корма.
9)	Присутствует блокировка на создание нового питомца сразу после смерти предыдущего.
10)	Все данные, необходимые для обработки находятся в конфигурируемом XML файле.
11)	Парсинг XML файла и вся работа с ним проводится без подключения сторонних библиотек, с помощью разработанных для этого классов, использующих только «поставляемые» библиотеки.

Следующие интервалы были выбраны с целью оптимального времени прохождения жизненного цикла приложения:
1)	Потеря шкалы «здоровья» и «веселья» - 5%/мин. (100% здоровья и настроения истратится без вмешательства за 20 минут)
2)	Блокировка на создание нового питомца после смерти старого – 3 мин.
3)	Каждый акт кормления или игры с питомцем прибавляет 5% к соответствующей шкале.
4)	Перерисовка GUI (конкретно шкал здоровья и настроения) осуществляется раз в 30 секунд.

Сборка проекта:
Сборка проекта осуществлялась при помощи СКВ Maven. В корневом каталоге проекта уже находится «pom.xml» файл со всеми необходимыми для сборки инструкциями. Структура каталога адаптирована под сборку:

<project>
    	--src
	         ---main
		     ----java
		     ----resources
<project>pom.xml
<project>target

Для того, чтобы собрать проект, необходимо:
1)	Открыть CMD и пройти в корневую директорию проекта.
2)	Воспользовавшись готовой инструкцией в pom.xml файле выполнить в CMD команду: mvn compile
3)	После окончания новой сборки проекта, запустить его можно командой mvn exec:java -Dexec.mainClass="sample.Main"

Изначально, проект уже собран, поэтому можно пропустить пункты 1 и 2, и сразу переходить в пункту 3.
